## 逻辑理解说明

根据示例推演，**"最大差值"是相对于当前分组的第一个元素（组首/基准值）**，而非相邻元素。这是一种"窗口锚定"策略：每组的基准固定为该组第一个进入的元素，后续元素都与这个基准比较。

---

```javascript
/**
 * 基于最大差值的分组归并函数
 * 分组策略：每个元素与其所在组的【第一个元素（基准值）】比较差值
 * 若差值超过阈值，则开启新分组，当前元素成为新基准
 *
 * @param {Array} arr - 输入数组（可包含稀疏项、null、undefined、非数字）
 * @param {number} threshold - 差值阈值（含等于，即 <= threshold 则归入当前组）
 * @returns {Array[]} 二维数组，每个子数组为一个分组
 */
function groupByMaxDiff(arr, threshold) {
  // ─── Step 1: 数据清洗 ───────────────────────────────────────────────────
  // 过滤掉 empty slot / null / undefined / 非数字，再升序排序
  const cleaned = Array.from(arr)          // 展开稀疏数组，empty slot → undefined
    .filter(item => item !== null && item !== undefined && typeof item === 'number' && !isNaN(item))
    .sort((a, b) => a - b);                // 升序排列

  // ─── 边界处理 ───────────────────────────────────────────────────────────
  if (cleaned.length === 0) return [];
  if (cleaned.length === 1) return [[cleaned[0]]];

  // ─── Step 2: 分组归并 ───────────────────────────────────────────────────
  const result = [];        // 最终二维结果数组
  let currentGroup = [];    // 当前正在构建的分组
  let anchor = null;        // 当前分组的基准值（组首元素）

  for (const num of cleaned) {
    if (currentGroup.length === 0) {
      // 当前分组为空，直接将该元素作为新基准
      anchor = num;
      currentGroup.push(num);
    } else if (num - anchor <= threshold) {
      // 与基准的差值 ≤ 阈值：归入当前分组
      currentGroup.push(num);
    } else {
      // 与基准的差值 > 阈值：封存当前分组，开启新分组
      result.push(currentGroup);
      anchor = num;           // 新基准 = 当前元素
      currentGroup = [num];   // 新分组以该元素起始
    }
  }

  // 别忘了把最后一个分组推入结果
  if (currentGroup.length > 0) {
    result.push(currentGroup);
  }

  return result;
}


// ═══════════════════════════════════════════════════════════════
//  测试验证
// ═══════════════════════════════════════════════════════════════

// ── 测试 1：题目示例 ──────────────────────────────────────────

console.log('测试1 输入:', input1);
console.log('测试1 输出:', JSON.stringify(result1));

// ── 测试 2：含稀疏/null/undefined 的脏数据 ───────────────────

console.log('\n测试2 含稀疏项输出:', JSON.stringify(result2));
console.log('测试2 通过:', JSON.stringify(result2) === '[[1,2,4,5,22,32,33],[56,67,68]]');

// ── 测试 3：未排序 + 混入非数字 ──────────────────────────────
const input3 = [68, null, 5, '文字', 1, undefined, 33, 22, 56, 4, NaN, 32, 2, 67];

console.log('\n测试3 脏数据+乱序输出:', JSON.stringify(result3));

// ── 测试 4：边界 - 空数组 ────────────────────────────────────

// ── 测试 5：边界 - 单元素 ────────────────────────────────────
```

## 核心逻辑总结

| 策略 | 说明 |
|------|------|
| **基准锚定** | 每组的比较基准是**组首元素**，固定不变 |
| **触发新组** | `当前元素 - 组首元素 > threshold` 时封存旧组，当前元素成新基准 |
| **含等于** | `<=` threshold 的差值仍归入当前组（33 - 1 = 32 ✅ 仍在组1） |
